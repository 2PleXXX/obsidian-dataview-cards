# **03 Логика секций**

## 🧠 Что такое `matchFactory`?

Каждая карточка попадает в определённую секцию на основе условия. Это условие определяется с помощью специального правила — `matchFactory`.  
Вы просто выбираете подходящий тип правила (один из доступных вариантов) и указываете нужные параметры в `matchArgs`.

---

## 📋 Обзор всех доступных правил (`matchFactory`)

### 🔹 `matchBySectionValue`

**Что делает:**  
Проверяет значение поля из YAML (указанного в `sectionGroupField` в конфиге).
В параметр `matchArgs` вы указываете одно или несколько значений, которые есть у поля `sectionGroupField`
Если в вашем YAML есть подходящее значение, то карточка попадет в соответствующую секцию.

**Когда использовать:**  
Если вы хотите разделить карточки по категориям (например: "SectionAudio", "SectionVideo").

**Пример:**

В одном YAML у вас будет:

```yaml
Section:
  - SectionPoster
```

В другом YAML:

```yaml
Section:
  - SectionAudio
```

В конфиге:

```js
//▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀
// 09 🧠 ЛОГИКА СЕКЦИЙ
//▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄
sectionGroupField: "Section",

//▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀
// 10 📦 СЕКЦИИ КАРТОЧЕК
//▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄
sections: [
{
  id: "section0",
  title: "Section Poster",
  matchFactory: "matchBySectionValue",
  matchArgs: ["SectionPoster"],
  fields: [
	"Poster",
	"Title",
	"Type",
	"tags",
  ],
},
{
  id: "section1",
  title: "Section Audio",
  matchFactory: "matchBySectionValue",
  matchArgs: ["SectionAudio"],
  fields: [
	"Audio",
	"Title",
	"Type",
	"tags",
  ],
},
],
```

### 🔹 `matchIfFieldEquals`

**Что делает:**  
Проверяет, равно ли значение определённого поля из YAML — заданному значению.  
Если совпадает, карточка будет отображена.

**Когда использовать:**  
Если вы хотите отобрать карточки, у которых в каком-либо поле указано конкретное значение.  
Например, только те, у кого поле `Favourite` — это `Yes`.

**Пример:**

```js
matchFactory: "matchIfFieldEquals",
matchArgs: ["Favourite", "Yes"]
```

	⚠️ Если в YAML указано несколько значений в поле, то достаточно, чтобы хотя бы одно значение совпадало. Например:

```yaml
Favourite:
  - None
  - Default
  - Yes
```

---
### 🔹 `matchIfFieldCompare`

**Что делает:**  
Сравнивает числовое значение из поля YAML (например, рейтинг) с заданным числом, используя условие (например, больше или равно).

**Какие типы принимает:**
Созданное поле в блоке `fields: [...]` должно иметь один из двух типов:
- "rating"
- "number"

**Когда использовать:**  
Если вы хотите отобрать карточки с рейтингом выше определённого значения. 

**Пример:**

```js
matchFactory: "matchIfFieldCompare",
matchArgs: ["Rating", 8, ">="]
```

**Варианты условий:**

```text
> — больше
>= — больше или равно
< — меньше
<= — меньше или равно
== — точно равно
!= — не равно
```

---
### 🔹 `matchIfContainsAllValues`

**Что делает:**  
Проверяет, содержат ли **любые поля** в карточке все указанные вами слова (значения).  
Если все указанные слова найдены хотя бы в каком-то поле — карточка будет отображена.

**Когда использовать:**  
Когда вы хотите вручную отобрать карточки, у которых встречаются конкретные слова, вне зависимости от того, в каком они поле.

**Пример:**

```js
matchFactory: "matchIfContainsAllValues",
matchArgs: ["Experimental", "Texture"]
```

> ✔️ Это универсальный способ фильтрации по словам. Подойдёт для поиска по жанрам, тегам, названиям и т.д.

---
### 🧾 Итог: для чего нужна каждая фабрика

|Название фабрики|Для чего нужна|
|---|---|
|`matchBySectionValue`|Проверяет значение определённого поля (указанного в `sectionGroupField`) и отображает карточку, если это значение входит в список заданных (`matchArgs`).|
|`matchIfFieldEquals`|Отображает карточку, если конкретное поле YAML содержит нужное значение. Подходит для отбора по одному признаку.|
|`matchIfFieldCompare`|Сравнивает числовое значение поля (например, рейтинг) с порогом. Работает с условиями вроде `больше`, `меньше`, `равно`.|
|`matchIfContainsAllValues`|Проверяет, встречаются ли указанные вами слова **где-либо** в карточке — в любом поле. Если все слова найдены, карточка отобразится.|

---

Эти фабрики работают **независимо друг от друга**. Вы можете использовать любую из них — одну, несколько или все сразу, в разных секциях.  
Скрипт просто применяет правило к каждой карточке и решает, показывать её в секции или нет. Всё остальное — ваш выбор.
