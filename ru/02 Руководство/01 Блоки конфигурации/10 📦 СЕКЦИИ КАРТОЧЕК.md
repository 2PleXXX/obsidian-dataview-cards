# БЛОК 10 📦 СЕКЦИИ КАРТОЧЕК

> 💡 Секции позволяют группировать карточки по определённым условиям. Например: "Смотрю", "Пройдено", "Избранное", и т.д.

---

## 🔧 Пример конфигурации секции:

```js
{
  id: "section0",
  title: "Section Poster",
  styleClass: "",
  matchFactory: "matchBySectionValue",
  matchArgs: ["SectionPoster"],
  fields: [
    "Poster",
    "Title",
    "DemoDate",
    "DemoList",
    "Badge",
    "DemoNum",
    "Type",
    "tags",
  ],
},
```

---

## 🔹 `id`

Уникальный идентификатор секции.  
— **Обязателен**.  
— Не должен повторяться с другими секциями.

> ❗ Специальные секции (`favourite`, `unsorted`) имеют зарезервированные `id`, их **нельзя переиспользовать**.

---

## 🔹 `title`

Заголовок секции, который будет отображён в интерфейсе.

- Пример: `"Section Poster"`

---

## 🔹 `styleClass`

Позволяет задать индивидуальный стиль для всей секции.  
Если не указан — используется стиль по умолчанию.

---

## 🔹 `matchFactory`

Отвечает за **логику**, по которой карточка попадает в секцию.

Наиболее часто используется:

```js
matchFactory: "matchBySectionValue",
```

Совместно с `matchArgs` определяет, какие карточки отбирать.

## 🔹 `matchArgs`

Параметры для `matchFactory`.  
Формат зависит от фабрики:

- Для `matchBySectionValue` — массив из одного или нескольких значений:


```js
matchArgs: ["SectionPoster"],
```

Здесь вы указываете то значение, которое указано в вашем YAML поле, которое задано в функции "sectionGroupField" (смотреть [[09 🧠 ЛОГИКА СЕКЦИЙ]]). 

Более подробно о каждой доступной логике вы можете прочитать [[03 Логика секций|Здесь]]

---

## 🔹 `fields`

Список полей, которые должны отображаться в карточке этой секции.

- Указываются по имени из `config.fields` (смотреть [[08 🖼 ОТОБРАЖАЕМЫЕ ПОЛЯ]])
    
- Отображаются **в том порядке**, в котором ты их перечислил
    

---

## ⭐ Специальные секции

### 🔸 Избранное (`favourite`)

```js
{
  id: "favourite",
  title: "⭐ Favourite",
  styleClass: "universal-favourite",
  allowDuplicates: true,
  matchFactory: "matchIfFieldEquals",
  matchArgs: ["Favourite", "Yes"],
  fields: [ ... ],
}
```

#### 🔹 `allowDuplicates`

Позволяет карточке одновременно быть в нескольких секциях.

- `true` — карточка может отображаться здесь **и** в других секциях
    
- `false` — карточка отобразится **только здесь**
    

Параметр `allowDuplicates = true` можно применять к абсолютно любой секции, где Вы хотите видеть дублирование карточек. Исключением является специальная секция "Не отсортированные".

> 📝 **Пример:** если карточка отмечена как `Favourite: Yes`, но уже попала в секцию "Смотрю", то при `allowDuplicates: false` она **не попадёт** в "Избранное".

---

## 🔸 Несортированные (`unsorted`)

```js
{
  id: "unsorted",
  title: "🗂 Несортированные",
  styleClass: "universal-unsorted",
  fields: [ ... ],
}
```

Секция, куда попадают **все карточки, не подошедшие ни под одну другую секцию**.
Я настоятельно рекомендую не удалять эту секцию, так как могут начаться сбои

> 📝 **Совет:** всегда размещайте секцию `unsorted` в самом конце — она служит «свалкой» для всех нераспознанных карточек.

---

## 💡 Общие советы

- Секции **обрабатываются сверху вниз**, в порядке указания в массиве `config.sections`.
    
- Чтобы карточка **не потерялась**, обязательно либо:
    
    - правильно настроить `matchFactory + matchArgs`, либо
        
    - оставить секцию `unsorted` для автоматического сбора.
        

---

## 🧠 Связь с блоком "[[09 🧠 ЛОГИКА СЕКЦИЙ]]"

В конфигурации Вы указываете, **какое поле из YAML использовать для группировки** (например, `sectionGroupField: "Section"`). Это поле участвует в `matchBySectionValue`.

Пример:

```yaml
Section: "SectionPoster"
```

→ попадёт в секцию с `matchArgs: ["SectionPoster"]`.
